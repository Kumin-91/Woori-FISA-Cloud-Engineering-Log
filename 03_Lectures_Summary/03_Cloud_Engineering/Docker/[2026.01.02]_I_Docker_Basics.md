# Docker Basics

## 1. Docker 설치 및 권한 설정

리눅스에 도커 엔진을 설치하고, 원활한 사용을 위해 권한을 조정합니다.

### 1.1 패키지 업데이트

```Bash
sudo apt update
```

### 1.2 Docker 설치 (simple 방법)

```Bash
sudo apt install -y docker.io
```

### 1.3 Docker 설치 확인

```Bash
docker --version
```

```Plain text
Docker version 28.2.2, build 28.2.2-0ubuntu1~24.04.1
```

### 1.4 현재 사용자 `docker` 그룹에 추가

* 일반 사용자가 sudo 없이 명령어를 사용할 수 있도록 설정

* 현재 접속 계정 (`$USER`)을 docker 그룹에 포함

```Bash
sudo usermod -aG docker $USER
```

* 그룹 설정 즉시 반영

```Bash
newgrp docker
```

### 1.5 Docker 실행 상태 확인

```Bash
sudo systemctl status docker
```

```Plain text
● docker.service - Docker Application Container Engine
     Loaded: loaded (/usr/lib/systemd/system/docker.service; enabled; preset: enabled)
     Active: active (running) since Sun 2026-01-04 20:35:16 KST; 33min ago

       Docs: https://docs.docker.com
   Main PID: 935 (dockerd)
      Tasks: 10
     Memory: 92.3M (peak: 93.6M)
        CPU: 899ms
     CGroup: /system.slice/docker.service
             └─935 /usr/bin/dockerd -H fd:// --containerd=/run/containerd/containerd.sock
````

* `Active: active (running)` 상태가 아니라면 아래 명령어로 서비스를 실행합니다.

```Bash
#서비스 즉시 실행
sudo systemctl start docker
#시스템 재부팅 시 서비스 자동 실행 활성화
sudo systemctl enable docker
```

## 2. Docker 명령어 가이드

Docker의 작업 흐름(이미지 다운로드 → 컨테이너 실행 → 관리)에 따른 명령어 모음입니다.

## 2.1 Image 관리

* 이미지 목록 확인

```Bash
docker images
```

* 이미지 다운로드

```Bash
docker pull <IMAGE>
```

* 이미지 삭제

```Bash
docker rmi <IMAGE>
```

## 2.2 Container 관리

* 컨테이너 생성 및 실행

```Bash
docker run -d --name <NAME> -p <HOST_PORT>:<CON_PORT> <IMAGE>
# -d : 백그라운드(데몬) 모드로 실행
# --name : 컨테이너에 고유 이름 부여
# -p : 호스트와 컨테이너 간의 포트 연결
```

* Container 중지

```Bash
docker stop <CONTAINER_ID | CONTAINER_NAME>
```

* Container 재시작

```Bash
docker start <CONTAINER_ID | CONTAINER_NAME>
```

* Container 삭제

```Bash
docker rm <CONTAINER_ID | CONTAINER_NAME>
```

### 2.3 Container 모니터링 및 내부 진입

* 실행 중인 Container

```Bash
docker ps
```

* 중지된 것을 포함한 모든 Container
```Bash
docker ps -a
```

* 상세 정보 조회

```Bash
docker inspect <CONTAINER_ID | CONTAINER_NAME>
```

* 실시간 로그 확인

```Bash
docker logs -f <CONTAINER_ID | CONTAINER_NAME>
# -f : 실시간(follow) 스트리밍
```

* 컨테이너 내부 진입

```Bash
docker exec -it <CONTAINER_ID | CONTAINER_NAME> bash
# 진입 후 OS 정보 확인 예시: cat /etc/os-release
```
