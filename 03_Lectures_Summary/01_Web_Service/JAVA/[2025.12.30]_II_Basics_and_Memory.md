# 변수의 생성/소멸 시점 & 참조 타입의 특징

## 1. 선언 위치에 따른 변수 분류

변수는 선언된 위치에 따라 메모리에 머무는 시간 (Lifecycle)과 접근 범위 (Scope)가 결정됩니다.

### 1.1 멤버 변수 (Member Variable)

클래스 (`class {}`) 내부에 선언되며, 객체의 구성 요소를 정의합니다.

* 저장 영역: Heap(힙) 메모리에 객체와 함께 저장됩니다.

* 생성/소멸: 객체가 생성될 때(new) 생성되고, 객체가 메모리에서 소멸(GC에 의해 삭제)될 때 함께 사라집니다.

* 초기화: 값을 지정하지 않으면 기본 데이터값으로 자동 초기화됩니다.

* 멤버 변수의 세부 구분

    * Instance 변수: 객체 생성이 필수인 변수로, 각 객체마다 독립적인 값을 가집니다.

    * Static 변수: 객체 생성과 무관하게 사용 가능한 변수로, 모든 객체가 공유합니다. (Method 영역 저장)

| 종류 | 기본 데이터값 |
| :---: | :---:|
| 정수 | 0 |
| 실수 | 0.0 |
| char | '\u0000' (null) |
| 논리 | false |
| 참조타입 | null (참조할 주소가 없음) |

### 1.2 로컬 변수 (Local Variable)

메소드, 생성자, 또는 특정 블록 (`()`, `{}`) 내부에 선언되는 변수입니다.

* 저장 영역: Stack(스택) 메모리에 저장됩니다.

* 생성/소멸: 메소드나 생성자가 호출될 때 생성되고, 실행이 종료될 때 메모리에서 즉시 자동 반환됩니다.

* 자동으로 초기화되지 않으므로 사용 전 반드시 수동으로 값을 할당해야 합니다.

## 2. 데이터 타입에 따른 구분

### 1.1 기본 타입 (Primitive Type, 원시타입)

JDK 내부에서 제공하는 내장 타입 (Build-in type)으로, 변수에 실제 값이 저장됩니다.

* 8가지의 타입 존재

| 카테고리 명칭 | 해당 타입 |
| :---: | :---:|
| 문자형 | char |
| 정수형 | byte, short, int, long |
| 실수형 | float, double |
| 논리형 | boolean |

### 2.2 참조 타입 (Reference Type)

기본 타입 이외의 모든 타입으로, 클래스를 기반으로 파생되는 무한대의 타입이 존재합니다.

* 데이터의 실체: 변수가 실제 보유한 데이터는 생성된 객체의 주소값입니다.

* 참조 변수를 출력하면 자동으로 toString()이 호출됩니다. 이는 최상위 클래스인 `java.lang.Object`가 모든 객체에 상속시키는 메소드입니다.

## 3. [용어 정리] 인스턴스(Instance)

* 정의: 클래스라는 설계도를 바탕으로 메모리에 실제로 생성되어 사용 가능한 실체화된 객체를 의미합니다.

* 생성 문법: `타입 변수명 = new 생성자()`;
