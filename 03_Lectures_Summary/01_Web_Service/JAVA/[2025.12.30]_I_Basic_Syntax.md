# JAVA 기본 문법 정리

## Types of Cases

| 표기법 | 자바에서의 주요 용도 | 예시 | 
| --- | --- | --- |
| camelCase | JAVA 변수, 메소드 | `userId`, `countWord()`
| snake_case | 자바보다 RDBMS | `user_id`, `app_status`
| UPPER_SNAKE_CASE | JAVA 상수 | `MAX_CNT`, `DEFAULT_PAGE`
| kebab-case | 사용하지 않음 | `user-id`, `user-address` |
| PascalCase | JAVA 클래스 | `MemberService`, `LoginController` |

## 자바 소스 파일 레이아웃

1. package 선언

2. import 선언

3. class 선언
    
    a. 변수 - option

    b. 생성자 - essential

    c. 메소드 - option

    d. static{} - option

```Java
package model;

import java.util.ArrayList;

public class SampleClass {
    //선언
	private ArrayList<String> messages;
    //기본 생성자
	public SampleClass() {}
    //생성자
	public SampleClass(ArrayList<String> messages) {
		super();
		this.messages = messages;
	}
    //메소드
	public ArrayList<String> getMessages() {
		return messages;
    }
    //메소드
	public void setMessages (ArrayList<String> messages) {
		this.messages = messages;
	}
    //static
	static {
		System.out.println("메모리에 로딩시 자동 실행되는 특화된 블록");
	}
}
```

## 클래스 작성 문법

### accessModifier: 접근 제어자 (접근 제한자)
    
* public, protected, (default), private

* 클래스, 변수, 생성자, 메소드 접근 범위 제한

* 적용

    * 클래스 선언: public 과 default 적용 가능

    * 변수, 생성자, 메소드, 선언: 전부 적용 가능

| 접근 제어자 | 동일 클래스 | 동일 패키지의 클래스 | 다른 패키지의 하위클래스 | 다른 패키지의 클래스 |
| :---: | :---: | :---: | :---: | :---: |
| public | ✅ | ✅ | ✅ | ✅ |
| protected | ✅ | ✅ | ✅ | ❌ |
| (defualt) | ✅ | ✅ | ❌ | ❌ |
| private | ✅ | ❌ | ❌ | ❌ |

### userModifier: 기타 제어자

* final, static, abstract, synchronized

* 클래스, 변수, 메소드, static 블록에 사용

* 적용

    * 클래스 선언: final, abstract

    * 변수 선언: final, static

    * 메소드 선언: final, static, abstract, synchronized

    * static 블록: static

| 기타 제어자 | 클래스 | 변수 | 생성자 | 메소드 | 블록 |
| :---: | :---: | :---: | :---: | :---: | :---: |
| final | ✅ | ✅ | ❌ | ✅ | ❌ |
| static | ❌ | ✅ | ❌ | ✅ | ✅ | 
| abstract | ✅ | ❌ | ❌ | ✅ | ❌ |
| synchronized | ❌ | ❌ | ❌ | ✅ | ✅ |

* **중첩 클래스(Inner Class)의 경우 static 선언이 가능, 최상위 클래스는 static 선언이 불가능**
